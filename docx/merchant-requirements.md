> **文档版本**：v1.0
> **编写日期**：2026-04-06
> **适用系统**：智慧旅游平台 商户端 Web 管理后台

## 一、产品定位与使用角色

### 1.1 产品定位

商户端（景点运营端）是智慧旅游平台的三端之一，面向**景区、博物馆、餐饮商户等运营方**，提供景点信息管理、门票/套餐管理、票务展示、评价管理、数据统计等核心功能。

商户端是商户与平台（政府端）、游客（用户端）之间的运营枢纽：

```
┌──────────────────────────────────────────────────────────────────┐
│                     智慧旅游平台                                    │
│                                                                │
│   ┌────────────┐   ┌──────────────────────────┐   ┌──────────┐│
│   │  用户端     │   │       商户端（本文档）    │   │  政府端   ││
│   │ 微信小程序  │◄──│    Vue3 Web 管理后台    │──►│ Web管理后台││
│   └────────────┘   └──────────────────────────┘   └──────────┘│
│         ▲                    │                       ▲          │
│         │                    │                       │          │
│         ▼                    ▼                       ▼          │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                  后端 REST API（Node.js + Express）       │  │
│   │          backend/src/modules/merchant/ 模块               │  │
│   └─────────────────────────────────────────────────────────┘  │
│                              │                                   │
│                              ▼                                   │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                MySQL 8.0（共用同一数据库）                 │  │
│   └─────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘
```

### 1.2 目标用户


| 用户类型       | 角色描述             | 典型场景               |
| ---------- | ---------------- | ------------------ |
| **景区运营商**  | 管理景区 POI 信息的运营人员 | 维护景点介绍、更新照片、设置开放时间 |
| **博物馆管理员** | 博物馆信息维护与讲解员      | 更新展品信息、管理票务     |
| **餐饮商户**   | 特色餐饮/咖啡店经营者      | 管理商户介绍、查看客流统计      |

> **注意**：商户与 POI 为**一对一同生关系**——商户注册账号时同步创建对应 POI（写入 `poi_info` + `merchant_poi_rel`，`is_primary = 1`）；POI 可由政府端/系统预设存在，但商户 POI 一旦存在即与商户一一绑定（`is_primary = 1`），商户的全部业务操作均围绕其关联的唯一 POI 展开。

### 1.3 与其他端的关系


| 关系方       | 数据交互                                | 说明                                                      |
| --------- | ----------------------------------- | ------------------------------------------------------- |
| **用户端**   | POI 信息、评论列表、评价回复                    | 商户修改 POI 信息后提交政府审核；回复用户评价                               |
| **政府端**   | POI 审核队列、商户入驻审核                     | 商户提交的信息进入 `poi_audit_queue`；政府审核通过后生效                   |
| **共用数据库** | poi_info、comment_info、ticket_info 等 | 商户端与政府端通过 `merchant_poi_rel`、`merchant_poi_review` 等表关联 |

## 二、核心业务流程

### 2.1 入驻认证流程

> **设计原则**：商户与 POI 为一对一同生关系——商户注册账号时同步创建对应 POI；政府端预置的 POI 由政府添加，商户接管/认领后形成关联。

```
商户提交入驻申请
      │
      ▼
填写基本信息（商户名称、手机号、密码、联系人、邮箱）
      │
      ▼
上传营业执照（可选字段）
      │
      ▼
选择经营类目（从 tag_info 表读取景区/博物馆/餐饮等标签）
      │
      ▼
写入 merchant 表（status = 0，待审核）
写入 merchant_review 表（action = "submit"）
同时新建关联 POI：写入 poi_info（status = 0，created_by = merchant.id）
写入 poi_audit_queue（submitter_type = "merchant"）
写入 merchant_poi_rel（is_primary = 1，poi_id → 刚创建的 poi_info.id）
      │
      ▼
政府端审核商户入驻（通过 → merchant.status → 1；驳回 → status = 3 + reject_reason）
      │
      ▼
商户端「景点信息管理」模块开放，可编辑已创建的 POI 详情（提交后进入 POI 审核队列）
```

**商户端 status 状态机**：


| status | 状态       | 可操作            |
| ------ | -------- | -------------- |
| 0      | 待审核      | 只能查看，不能管理      |
| 1      | 审核通过（正常） | 全功能可用          |
| 2      | 封禁       | 禁止登录，展示封禁原因    |
| 3      | 驳回       | 只能查看驳回原因，可重新提交 |
| 4      | 注销       | 禁止登录，数据保留      |


### 2.2 景点信息维护流程

> **前置条件**：商户已完成入驻审核（merchant.status = 1），尚未提交 POI 或 POI 正在审核/已上线。

```
商户登录（手机号 + 密码，JWT 认证）
      │
      ▼
进入「景点管理」页面
      │
      ├── 尚未提交 POI → 填写 POI 信息（poi_info 全量字段）→ 提交审核
      │                                              │
      │                                              ▼
      │                                    写入 poi_info（status = 0）
      │                                    写入 poi_audit_queue
      │                                    写入 merchant_poi_rel（is_primary = 1）
      │
      └── 已有关联 POI → 点击「编辑景点」→ 修改 POI 字段 → 提交变更
                                                      │
                                                      ▼
                                           写入 merchant_poi_review（记录变更）
                                           写入 poi_audit_queue（关键字段变更需重审）
```

### 2.3 门票/票务管理流程

> 商户关联唯一一个 POI，因此所有票务均属于该 POI。

```
商户在「票务管理」页面管理关联 POI 的票种
      │
      ▼
查看该 POI 的票种列表（来自 ticket_info 表，poi_id = 商户主 POI）
      │
      ├── 新增票种（ticket_name、description、price、stock）
      │         │
      │         ▼
      │    ticket_info 表新增记录（status = 1，在售）
      │
      ├── 编辑票种（修改价格、库存、描述）
      │
      └── 下架票种（status → 0，不在售）
```

### 2.4 评价管理流程

```
用户端在 POI 详情页发表评论（POST /api/comment/create）
      │
      ▼
评论写入 comment_info 表（status = 1，audit_status = 0）
      │
      ▼
商户端「评价管理」页面展示该 POI 下的所有评论
      │
      ▼
商户可对评论进行回复（merchant_reply 字段）
      │
      ▼
回复写入 comment_info.merchant_reply（由商户侧更新）
```

## 三、功能模块清单

### 模块 A：入驻与认证

#### A-1. 注册账号

| 子功能        | 功能说明                                       | 数据来源                       |
| ---------- | ------------------------------------------ | -------------------------- |
| 手机号 + 密码注册 | 商户填写商户名称、手机号、密码、联系人、邮箱后提交注册                | merchant 表                 |
| 经营类目选择     | 注册时从 tag_info 表读取商户类别（景区 / 博物馆 / 餐饮 等）      | tag_info.category = "merchant_category" |
| 营业执照上传     | 上传营业执照图片（调用统一上传接口 POST /api/common/upload） | merchant.business_license  |
| 注册状态查询     | 注册后可查询当前审核状态（待审核 / 审核通过 / 驳回）              | merchant.status            |

> **商户注册时同步创建 POI**：注册账号时系统自动创建对应 POI 记录（poi_info + merchant_poi_rel，is_primary=1），初始状态为待审核。注册表单填写的信息（名称、地址、联系方式等）将作为 POI 字段的初始值。POI 完整字段（description、photos、opening_time 等）需在后续「景点信息管理」模块中编辑补充。

**注册字段约束**：

| 字段                | 约束           | 对应 poi_info 字段 | 说明          |
| ----------------- | ------------ | -------------- | ----------- |
| merchant_name     | 必填，2-100字    | poi_name（中文）  | 商户名称（也是景点名称） |
| tel               | 必填，唯一，11位手机号 | tel            | 登录账号 / 景点电话 |
| password          | 必填，6位以上      | —              | bcrypt 加密存储 |
| contact_person    | 选填，2-50字     | —              | 联系人姓名       |
| email             | 选填，有效邮箱格式    | email          | 接收通知        |
| merchant_category | 必填，枚举值       | tag_info 关联     | 从 tag_info 表读取，category="merchant_category" |
| business_license  | 选填，URL       | —              | 需通过图片安全审核   |
| address           | 选填           | address（中文）   | 商家/景点地址     |

**经营类目来源（tag_info 表）**：

| tag_code  | 标签名称（中文） |
| -------- | --------- |
| scenic     | 景区景点 |
| museum     | 博物馆  |
| restaurant | 餐饮   |
| food       | 特色美食 |
| cafe       | 咖啡茶饮 |
| street     | 特色街区 |
| theme_park | 主题乐园 |
| park       | 公园绿地 |
| other      | 其他   |

> 经营类目从 `tag_info` 表中 `category = "merchant_category"` 的标签中选取，`tag_code` 即为枚举值。商户入驻后可在「景点信息管理」中通过 `poi_tag_rel` 表关联更多主题标签（如历史、建筑、博物馆等）。


#### A-2. 登录与认证


| 子功能        | 功能说明                                       |
| ---------- | ------------------------------------------ |
| 手机号 + 密码登录 | 商户使用注册手机号和密码登录                             |
| JWT Token  | 登录成功后返回 JWT Token，有效期 7 天，存储在 localStorage |
| Token 刷新   | Token 即将过期时自动刷新                            |
| 记住登录       | 提供"记住我"选项，最长 30 天                          |
| 退出登录       | 清除 Token，返回登录页                             |


**登录接口**：复用后端现有 `merchant` 模块接口，认证方式与政府端 `government` 一致（手机号 + 密码 + JWT）。

#### A-3. 账号找回


| 子功能   | 功能说明                                            |
| ----- | ----------------------------------------------- |
| 忘记密码  | 输入手机号 + 短信验证码，重置密码                              |
| 发送验证码 | POST /api/user/send-code（type = "reset"），60 秒冷却 |
| 重置密码  | POST /api/user/reset-password，验证通过后更新密码         |

### 模块 B：景点信息管理

#### B-1. 景点信息概览

> 商户与 POI 为一对一同生关系——商户注册账号时同步创建对应 POI；POI 可由政府端/系统预设存在，但商户 POI 一旦存在即与商户一一绑定（is_primary = 1）。进入「景点管理」页面直接展示该 POI 详情，而非列表。

| 子功能     | 功能说明                                          | 数据来源                           |
| ------- | --------------------------------------------- | ------------------------------ |
| POI 状态展示 | 展示当前商户关联的 POI 状态（待审核 / 已上线 / 已下架）         | poi_info.status                |
| 基本信息卡片  | 景点名称、地址、联系方式、简介（多语言）展示                  | poi_info 相关字段               |
| 统计数据卡片  | 累计打卡次数、平均评分、评论数量、收藏数量                  | poi_info + poi_stats           |
| 审核进度查看  | 如 POI 正在审核，显示审核队列中的位置和预计等待时间             | poi_audit_queue               |
| 快捷操作入口  | 「编辑景点」「查看审核历史」「管理票种」等快捷入口              | —                              |

**概览字段**：

| 字段            | 说明                   |
| ------------- | -------------------- |
| poi_name      | 景点名称（多语言，取当前语言）      |
| address       | 地址（多语言）               |
| tel           | 联系电话                  |
| district      | 行政区                  |
| status        | 状态：0 待审核，1 已上线，2 已下架 |
| check_count   | 累计打卡次数               |
| avg_rating    | 平均评分                 |
| comment_count | 评论数量                 |
| is_free       | 是否免费                 |
| updated_at    | 最后更新时间               |


#### B-2. 编辑景点

> **逻辑变更**：商户 POI 在注册账号时同步新建（状态为待审核），后续不再有「新增景点」入口，只能进入编辑表单修改 POI 字段并提交审核。

| 子功能    | 功能说明                                       |
| ------ | ------------------------------------------ |
| 基础信息填写 | POI 名称（多语言 JSON）、描述（多语言 JSON）、地址（多语言 JSON） |
| 地理位置   | 经纬度（支持地图选点）、行政区（自动解析或手动选择）                 |
| 联系方式   | 电话、邮箱、官方网址                                 |
| 媒体信息   | 照片列表（最多 20 张，支持拖拽排序，第一张为封面）                |
| 基本属性   | 是否免费、是否需要门票、是否需要预约                         |
| 标签管理   | 从 tag_info 选择标签（支持多选，设置权重 0-1）             |
| 开放时间   | 从 opening_time 规则中新增/编辑开放时间规则              |
| 提交审核   | 提交后进入 poi_audit_queue，等待政府端审核              |

**多语言字段格式**（仅支持 zh 与 en 两种语言）：

```json
{
  "zh": "景点名称",
  "en": "Attraction Name"
}
```

#### B-3. 上下架管理

| 子功能    | 功能说明                                                           |
| ------ | -------------------------------------------------------------- |
| 下架景点   | 商户主动下架（poi_info.status → 2），无需审核，立即生效                          |
| 上架景点   | 下架后重新上架视为信息变更，需提交审核                                            |
| 政府强制下架 | 政府端可直接将 poi_info.status → 2（通过 poi_audit_queue 或直接操作），商户收到系统通知 |

### 模块 C：门票/套餐管理

#### C-1. 票种列表

> 商户只有一个 POI，所有票种均属于该 POI，无需按景点筛选。

| 子功能   | 功能说明                            | 数据来源                                      |
| ----- | ------------------------------- | ----------------------------------------- |
| 票种列表  | 展示商户 POI 的所有 ticket_info 记录 | ticket_info（poi_id = 商户主 POI）         |
| 按状态筛选 | 全部 / 在售 / 停售                    | ticket_info.status                        |
| 排序    | 按创建时间 / 价格排序                    | ticket_info.createdAt / ticket_info.price |


**列表字段**：


| 字段          | 说明           |
| ----------- | ------------ |
| poi_name    | 所属景点名称       |
| ticket_name | 票种名称（多语言）    |
| price       | 价格           |
| stock       | 余票           |
| status      | 状态：0 停售，1 在售 |
| created_at  | 创建时间         |


#### C-2. 新增票种


| 子功能  | 功能说明                  |
| ---- | --------------------- |
| 票种名称 | 多语言 JSON，必填           |
| 票种描述 | 多语言 JSON，选填           |
| 价格   | DECIMAL(10,2)，必填，≥ 0  |
| 库存数量 | INT，必填，≥ 0；设为 0 表示不限量 |
| 状态   | 默认在售（status = 1）      |


#### C-3. 编辑/停售票种


| 子功能    | 功能说明                           |
| ------ | ------------------------------ |
| 修改价格   | 直接修改，立即生效                      |
| 修改库存   | 直接修改，立即生效                      |
| 修改票种信息 | 修改名称/描述，立即生效                   |
| 停售     | status → 0，立即生效 |
| 恢复销售   | status → 1，立即生效                |


#### C-4. 票务统计概览

| 子功能   | 功能说明                                |
| ----- | ----------------------------------- |
| 票种总数 | 当前 POI 票种数量                       |
| 在售票种  | 状态为在售（status=1）的票种数量            |
| 停售票种  | 状态为停售（status=0）的票种数量            |
| 余票预警  | 库存 ≤ 10 张时高亮提示                      |


### 模块 D：消息通知

> 商户端消息通知模块用于接收系统公告、用户互动提醒及政府端通知，帮助商户及时掌握运营动态。

#### D-1. 消息中心

| 子功能   | 功能说明                                          | 数据来源        |
| ----- | --------------------------------------------- | ----------- |
| 消息列表  | 展示商户所有消息（系统公告 / 用户互动 / 政府通知）按时间倒序 | message_info |
| 消息分类  | 全部 / 系统公告 / 互动提醒 / 政府通知                 | message_info.type |
| 未读红点  | 顶部导航栏显示未读消息数量，有新消息时实时推送             | message_info.is_read |
| 标记已读  | 单条点击标记已读，或"全部已读"批量操作                  | message_info.is_read |
| 消息详情  | 点击消息卡片展开详情，含时间、来源、内容                   | message_info |

**消息类型定义**：

| type 值 | 类型       | 说明               |
| ----- | -------- | ---------------- |
| system | 系统公告   | 平台运营通知、版本更新等   |
| user   | 用户互动   | 用户新评论、新收藏等提醒   |
| gov    | 政府通知   | 审核结果、下架通知等     |
| audit  | 审核进度   | POI/票务审核状态变更通知  |

**消息字段**：

| 字段       | 说明                    |
| -------- | --------------------- |
| title    | 消息标题                  |
| content  | 消息内容（超过 200 字折叠）      |
| type     | 消息类型                  |
| is_read  | 是否已读                  |
| related_id | 关联业务 ID（POI / 评论等）  |
| created_at | 发送时间                 |

#### D-2. 消息推送规则

| 触发场景                | 推送内容                    | 接收方   |
| ------------------- | ----------------------- | ----- |
| 用户评论商户 POI         | "用户 {nickname} 评论了您的景点"   | 商户端  |
| 商户 POI 审核结果下达     | "您的景点「{poi_name}」审核{result}" | 商户端  |
| 政府强制下架 POI         | "景点「{poi_name}」已被下架，原因：{reason}" | 商户端  |
| 系统公告 / 运营通知       | 平台公告全文                   | 所有商户端 |
| 用户收藏 / 点赞商户 POI   | "用户 {nickname} 收藏了您的景点"   | 商户端  |


### 模块 E：营销管理

> 商户可管理营销活动、优惠券等，提升景点曝光与转化。

#### E-1. 活动管理

| 子功能     | 功能说明                              |
| ------- | --------------------------------- |

### 6.2 客流分析（POI Stats 模块）

#### 6.2.1 打卡趋势

| 子功能        | 功能说明                       |
| ---------- | -------------------------- |
| 近 30 天打卡趋势 | 折线图，横轴为日期，纵轴为打卡次数，每日一个数据点 |
| 近 7 天对比     | 与上周同期对比，用不同颜色折线叠加显示         |
| 高峰日标注      | 打卡数最高的 3 天高亮标记，并显示具体数值         |
| 时间段分布      | 柱状图，横轴为小时段（0-23），纵轴为打卡次数     |

**数据来源**：poi_stats 表（date, check_count）+ check_info 表（hour_tag 聚合）

**SQL 示例**：

```sql
-- 打卡趋势
SELECT date, check_count
FROM poi_stats
WHERE poi_id = :poiId AND date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
ORDER BY date ASC;

-- 时段分布
SELECT hour_tag, COUNT(*) as count
FROM check_info
WHERE poi_id = :poiId AND check_time >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY hour_tag;
```

| 总评价数     | COUNT(id) WHERE poi_id = 商户POI     |

## 七、异常与边界场景

### 7.1 入驻与认证异常


| 场景              | 处理方式                                         |
| --------------- | -------------------------------------------- |
| 手机号已被注册         | 提示"该手机号已注册"，引导找回密码或联系客服                      |
| 营业执照上传失败        | 提示上传失败原因（格式错误/大小超限/图片不合规），允许重新上传             |
| 注册后被驳回          | 在登录页展示驳回原因（merchant.reject_reason），允许修改后重新提交 |
| 账号被封禁（status=2） | 禁止登录，提示"账号已被封禁，原因：{reason}"，引导联系平台客服         |
| Token 泄露        | 登录页提供"账号被盗"入口，可强制下线所有设备（清除所有 Token）          |


### 7.2 景点管理异常


| 场景               | 处理方式                                         |
| ---------------- | -------------------------------------------- |
| 同一 POI 被多家商户申请关联 | 第一家审核通过后，其他申请自动驳回（提示"该景点已被认领"）               |
| 景点审核被政府拒绝        | 展示拒绝原因（来自 poi_audit_queue.remark），商户可修改后重新提交 |
| 删除景点时存在关联票务数据
| 地图选点超出平台区域范围     | 提示"请选择平台服务范围内的位置"（仅限上海市）                     |


### 7.3 票务管理异常


| 场景           | 处理方式                         |
| ------------ | ---------------------------- |
| 库存为 0 时用户仍下单 | 下单前校验库存；库存为 0 时前端禁止购买，后端再次校验 |
| 修改价格时存在已购用户
| 删除在售票种       | 允许删除（status → 0），已购用户不受影响  |
| 余票预警         | 库存 ≤ 10 张时，在票种列表该行高亮红色警告标记   |


### 7.5 系统级异常


| 场景      | 处理方式                              |
| ------- | --------------------------------- |
| 后端服务不可用 | 展示"服务暂不可用，请稍后再试"，记录错误日志           |
| 图片上传失败  | 提示上传失败，允许重试；重试 3 次仍失败提示联系技术支持     |
| 短信发送失败  | 提示"验证码发送失败，请稍后重试"；60 秒冷却结束后可重新发送  |
| 数据库连接失败 | 503 Service Unavailable，记录错误日志并告警 |

### 8.2 新增表详细设计

#### 表 1：`merchant_employee`（商户员工表）

**用途**：支持商户创建子账号（管理员、票务员、查看员），实现权限分级管理。

```prisma
model MerchantEmployee {
  id           Int       @id @default(autoincrement())
  merchantId   Int       @map("merchant_id")
  tel          String    @unique @map("tel")
  password     String    @map("password")
  name         String    @map("name")
  role         Int       @default(0) @map("role")   // 0管理员 1票务员 2查看员
  status       Int       @default(1) @map("status")  // 1正常 0禁用
  lastLoginAt  DateTime? @map("last_login_at")
  createdAt    DateTime  @default(now()) @map("created_at")
  updatedAt    DateTime  @updatedAt @map("updated_at")

  merchant Merchant @relation(fields: [merchantId], references: [id], onDelete: Cascade)

  @@index([merchantId])
  @@index([tel])
  @@map("merchant_employee")
}
```

**字段说明**：


| 字段            | 类型           | 约束                        | 说明                |
| ------------- | ------------ | ------------------------- | ----------------- |
| id            | INT          | PK                        | 主键                |
| merchant_id   | INT          | NOT NULL, FK(merchant.id) | 所属商户              |
| tel           | VARCHAR(20)  | UNIQUE, NOT NULL          | 员工手机号（登录账号）       |
| password      | VARCHAR(255) | NOT NULL                  | bcrypt 加密密码       |
| name          | VARCHAR(50)  | NOT NULL                  | 员工姓名              |
| role          | INT          | DEFAULT 0                 | 角色：0管理员，1票务员，2查看员 |
| status        | INT          | DEFAULT 1                 | 状态：1正常，0禁用        |
| last_login_at | DATETIME     | NULL                      | 最后登录时间            |
| created_at    | DATETIME     | DEFAULT NOW               | 创建时间              |
| updated_at    | DATETIME     | ON UPDATE NOW             | 更新时间              |


---
```

#### 表 5（可选）：`merchant_stats`（商户统计表）

**用途**：按商户维度聚合每日统计数据，支撑 Dashboard 快速查询（避免跨 POI 实时聚合）。

```prisma
model MerchantStats {
  id           Int       @id @default(autoincrement())
  merchantId   Int       @map("merchant_id")
  createdAt    DateTime  @default(now()) @map("created_at")
  updatedAt    DateTime  @updatedAt @map("updated_at")

  merchant Merchant @relation(fields: [merchantId], references: [id], onDelete: Cascade)

  @@unique([merchantId, date])
  @@map("merchant_stats")
}
```

**说明**：每日凌晨由定时任务从 `check_info`、`comment_info` 等表聚合写入。商户 Dashboard 直接查询本表，避免大表实时聚合。

#### 表 6（可选）：`operation_log`（操作日志表）

**用途**：记录商户端关键操作日志，支持审计追溯。

```prisma
model MerchantOperationLog {
  id           Int       @id @default(autoincrement())
  merchantId   Int       @map("merchant_id")
  employeeId   Int?      @map("employee_id")             // 操作员工，NULL为主账号
  action       String    @map("action")                 // 操作类型：login/logout/create_poi/edit_poi/offline_poi/create_ticket/edit_ticket/reply_comment/...
  targetType   String?   @map("target_type")           // 操作对象类型：poi/ticket/order/comment/employee
  targetId    Int?      @map("target_id")             // 操作对象ID
  detail       Json?     @map("detail")                // 操作详情（变更前后数据）
  ip           String?   @map("ip")
  userAgent    String?   @map("user_agent")
  createdAt    DateTime  @default(now()) @map("created_at")

  merchant  Merchant  @relation(fields: [merchantId], references: [id], onDelete: Cascade)
  employee  MerchantEmployee? @relation(fields: [employeeId], references: [id], onDelete: SetNull)

  @@index([merchantId])
  @@index([employeeId])
  @@index([action])
  @@index([createdAt])
  @@map("merchant_operation_log")
}
```
