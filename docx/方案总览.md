# 智慧旅游平台方案总览

## 一、项目概述

智慧旅游平台是一个面向游客、商户、政府管理方的智能旅游服务系统，提供AI智能推荐、路线规划、地图打卡、内容分享、商户管理、政府监管等核心功能。

### 1.1 项目组成

| 端 | 技术栈 | 形态 |
|---|--------|------|
| 用户端 | uni-app + Vue3 + TypeScript | 微信小程序 |
| 后端服务 | Node.js + Express.js + Prisma | RESTful API |
| 商户端 | Vue3 + Element Plus + TypeScript | Web管理后台 |
| 政府端 | Vue3 + Element Plus + TypeScript | Web管理后台 |

### 2.3 核心功能流程

#### 流程A：冷启动个性化推荐

```
用户首次启动
     ↓
自动匿名登录 → 获取Token
     ↓
检查偏好记录 → 无记录 → 展示偏好收集弹窗（3步）
     ↓
提交偏好 → 首页展示个性化推荐
```

#### 流程B：AI路线生成

```
用户输入需求（文字/语音）≥10字
     ↓
调用 POST /api/route/generate
     ↓
后端：NLP解析 → 标签匹配 → 候选POI筛选 → LLM精选 → 路线数据
     ↓
前端展示路线详情（地图 + 按天分组）
     ↓
用户可：保存 / 编辑 / 打卡
```

#### 流程C：打卡前置评论

```
用户在路线详情页点击"打卡"
     ↓
GPS定位 → 计算与POI距离
     ↓
距离>200米 → 提示"距离较远，无法打卡"
     ↓
距离≤200米 → 写入打卡记录
     ↓
展示打卡成功动画
     ↓
引导进入评论发布页
     ↓
发表星级评分+文字评论+图片（可选）
```

### 3.5 认证体系

本系统采用**三级用户认证体系**：

| 层级 | 认证方式 | 可用功能 | Token有效期 |
|------|----------|----------|-------------|
| 公共接口 | 无需认证 | POI搜索/浏览、官方内容浏览 | - |

## 六、AI服务方案

### 6.1 AI能力概述

| 能力 | 技术方案 | 说明 |
|------|----------|------|
| 路线生成 | OpenAI GPT-4 | 根据用户需求生成结构化旅游路线 |
| 智能推荐 | 标签匹配 + GPT | 基于用户偏好推荐POI |
| 意图识别 | NLP解析 | 解析用户输入中的天数、偏好等关键信息 |

### 6.2 路线生成流程

```
1. 用户提交需求（文字描述）
        ↓
2. NLP解析：提取天数、偏好关键词、特殊需求
        ↓
3. 标签匹配：从 poi_tag_rel 表筛选候选 POI（≤50个）
        ↓
4. LLM处理：将候选 POI + 用户画像输入 GPT-4
        ↓
5. 生成路线：返回结构化路线数据（按天分组、包含建议时长）
        ↓
6. 保存路线：写入 routes_info 表
        ↓
7. 返回路线 ID 和详情
```

### 6.3 提示词设计

#### 系统角色提示词

```
你是一位专业的旅游规划师，为用户提供个性化的旅游路线规划服务。
你需要根据用户的需求和偏好，从景点列表中选择最合适的景点，
合理安排每天的行程，确保路线具有可行性和趣味性。
```

## 八、多语言支持方案

### 8.1 多语言字段存储

数据库中多语言字段采用 JSON 格式存储：

```json
{
  "zh": "景点名称",
  "en": "Attraction Name",
  "ja": "アトラクション名",
  "ko": "관광지 이름"
}
```

### 8.2 国际化实现

- 前端：根据用户语言设置，从 locales 文件加载对应语言包
- 后端：根据请求头 `Accept-Language` 返回对应语言内容
- 冷启动：根据国籍自动设置默认语言
